# 产品说明：微信读书笔记知识库

## 背景

每两周读一本书，在微信读书中积累划线和想法。但这些笔记散落在微信读书 App 里，**读完就忘，需要时找不到**。

## 核心痛点

**阅读积累无法复用**。花了时间读书、思考、划线，但当需要回忆某个观点时，经常因为找不到而放弃。相当于每次都在从零开始思考，之前的阅读投入被浪费了。

微信读书自带搜索只支持关键词精确匹配，无法满足"模糊回忆"场景——比如"我记得读过一个关于从基本事实出发推理的方法"，但想不起来"第一性原理"这个词。

## 使用场景

### 场景一：模糊检索（约每周一次）

> "我记得在哪本书里看过这个观点，但想不起来了"

- **工作中写方案**时，想引用某个理论或框架来支撑观点
- **跟人聊天**时，想举一个书里的例子但记不清出处
- **自己思考问题**时，隐约记得某本书讨论过类似话题

**需要能力**：AI 语义搜索，用自然语言描述就能找到相关笔记。比如问"和第一性原理相关的书有哪些"。

### 场景二：定期回顾

> "书是常读常新的，我想看看自己不同阶段的理解"

定期翻看自己的划线和想法，了解认知变化的轨迹。

### 场景三：写作与输出

写文章、发朋友圈、写周报时，快速找到可引用的金句或观点，并带有原书出处。

### 场景四：主题串联

多本书提到了同一个概念（如"认知偏差"），想把不同作者的观点放在一起对比，形成更完整的认识。

**需要能力**：按主题标签筛选，跨书籍查看同主题笔记。

### 场景五：推荐分享

朋友问"有没有关于 XX 的书推荐"，能快速找到自己读过的相关书和核心笔记。

### 场景六：复盘成长

半年或一年回看，发现自己关注的主题从技术转向了管理，能看到认知演变的轨迹。

## 目标

搭建一个**个人认知基础设施**：

1. 微信读书的划线和想法自动同步到 Notion
2. AI 自动为书籍打上主题标签，方便分类筛选
3. 利用 Notion AI 实现语义搜索，支持模糊检索
4. 越早建立越好——阅读量会持续增长（预计每年约 25 本），问题只会越来越严重

## 设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 笔记平台 | Notion | 用户已在使用；Notion AI 原生支持语义搜索 |
| 数据获取 | 微信读书非官方 API + Cookie | 无官方 API，这是所有开源项目验证过的方案 |
| 标签方式 | Claude Haiku 自动生成主题标签 | 按主题分类，成本低，支持中文 |
| 同步策略 | 增量同步（不删除已有内容） | 保留用户在 Notion 中的手动补充 |
| 运行方式 | GitHub Actions 每日定时 | 免费、不依赖本地电脑开机 |
| Cookie 管理 | 手动刷新（约1-2周一次） | 最稳定可靠，自动化登录容易失败 |
